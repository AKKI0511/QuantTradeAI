# Getting Started

This guide covers the implemented onboarding paths in QuantTradeAI.

## Quick Installation

### Prerequisites

- Python 3.11 or higher
- Poetry (recommended) or pip

### Install with Poetry

```bash
git clone https://github.com/AKKI0511/QuantTradeAI.git
cd QuantTradeAI
poetry install
```

### Install with pip

```bash
git clone https://github.com/AKKI0511/QuantTradeAI.git
cd QuantTradeAI
pip install -r requirements.txt
```

## First Steps

### 1. Generate a Canonical Project Config

```bash
poetry run quanttradeai init --template research -o config/project.yaml
poetry run quanttradeai validate -c config/project.yaml
```

Use this path when you want the canonical Stage 1 workflow. `config/project.yaml` is generated by the CLI and is not checked into the repository by default.

Supported starter templates are `research`, `llm-agent`, and `hybrid`.

### 2. Run the Canonical Research Workflow

```bash
poetry run quanttradeai research run -c config/project.yaml
```

This validates the project config, writes resolved runtime configs under `runs/<timestamp>/`, and stores training artifacts under `models/experiments/<timestamp>/`.

### 3. Fetch Data for the Legacy Multi-File Workflow

```bash
poetry run quanttradeai fetch-data
```

This downloads OHLCV data for the tickers listed in `config/model_config.yaml` and caches them locally.

### 4. Run the Legacy Training Pipeline

```bash
poetry run quanttradeai train
```

This will:

- Process the data and generate features
- Train ensemble models for each symbol
- Optimize hyperparameters automatically
- Save per-run artifacts under `models/experiments/<timestamp>/`

Tip: to control the test window, set `data.test_start` and optionally `data.test_end` in `config/model_config.yaml`. Hyperparameter tuning uses time-series cross-validation via `training.cv_folds`.

### 5. Evaluate Results

```bash
poetry run quanttradeai evaluate -m models/experiments/<timestamp>/<SYMBOL>
```

### 6. Backtest a Saved Model

```bash
poetry run quanttradeai backtest-model -m models/experiments/<timestamp>/<SYMBOL> \
  -c config/model_config.yaml -b config/backtest_config.yaml
```

This uses the saved model's `feature_columns` together with the execution config. Artifacts are written under `reports/backtests/<run_timestamp>/<SYMBOL>/`:

- `metrics.json` with gross and net performance metrics
- `equity_curve.csv` with the time series
- `ledger.csv` with per-trade fills and costs when trades occur

Portfolio-level aggregates are also written under `reports/backtests/<run_timestamp>/portfolio/`.

## Understanding the Output

After running the training pipeline, you will find:

- Saved models in `models/experiments/<timestamp>/<SYMBOL>/`
- Run-level metrics and validation artifacts in `models/experiments/<timestamp>/`
- Cached market data in `data/raw/`
- Feature-engineered outputs in `data/processed/`

## Configuration

QuantTradeAI supports two config entrypoints:

- `config/project.yaml` for `quanttradeai validate` and `quanttradeai research run`
- Legacy YAML files under `config/` for `train`, `evaluate`, `backtest-model`, and live trading

See [Configuration Guide](configuration.md) for details.

## Troubleshooting

### Data fetching fails

```bash
poetry run quanttradeai fetch-data --refresh
```

### Training takes too long

- Reduce the configured symbol list
- Reduce the optimization search space or trial count in the training code path you are using

### Memory issues

- Reduce the configured date range
- Process fewer symbols at once
